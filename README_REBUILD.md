# Metropy 리빌딩 프로젝트
**서울 지하철 착석 효용 의사결정 모델**

## 프로젝트 개요

이 프로젝트는 **예측이 아닌 의사결정** 문제로 접근합니다.
- 목표: "어느 차량(칸)에 탑승해야 착석 효용이 최대화되는가?"
- 방법론: 공공데이터 + 구조적 가정 기반 효용 점수 모델

## 핵심 전제

1. **착석 경험 > 빠른 하차**
   - 출구까지 10분 단축 < 이동 중 5분 착석
   - 이동 후 편의성보다 이동 중 체력 소모 최소화

2. **정답 데이터 없음**
   - 칸별 착석률, 칸별 하차 인원 정답 데이터는 존재하지 않음
   - 따라서 예측 문제가 아닌 점수 기반 의사결정 문제로 전환

3. **머신러닝의 역할**
   - 최종 목표가 아닌 SeatScore 계산을 위한 보조 모듈
   - 혼잡도 예측 → 하차 규모 근사 → 효용 점수 입력값 생성

## SeatScore 모델

### 정의
```
SeatScore(c) = Σ [ D(s) × T(s → destination) × w(c, s) ]
```

- **c**: 차량(칸) 번호
- **s**: 중간역
- **D(s)**: 해당 역의 하차 규모 근사 지표 (혼잡도 예측 모델 출력)
- **T(s → destination)**: 해당 역 이후 목적지까지 이동 시간
- **w(c, s)**: 구조적 하차 집중 가중치 (Fast Exit API 기반)

### 의미
- 하차가 많이 발생하는 역에서 빨리 좌석 확보 → 높은 효용
- 빠른 하차 칸에 가중치 부여 → 착석 가능성 상승
