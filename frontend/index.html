SeatScore v3 공식

SeatScore(c) = Σs [ D(s) × T(s→dest) × w(c,s) × α(h) / L(c) ] − β × B(c,h)
D(s) 방향별 혼잡도 하락량
경유역 s에서 빠지는 승객 규모
T(s→dest) 실제 잔여 이동 시간
TMAP 대중교통 API 기반 (초 단위)
w(c,s) 칸별 하차 가중치
SK 지하철 혼잡도 API 칸별 하차율 기반
α(h) 시간대 배율 (연속 보간)
출근러시 1.4 / 퇴근 1.3 / 심야 0.6
L(c) 탑승 경쟁 계수
같은 칸 탑승자와의 좌석 경합 반영
β × B(c,h) 탑승 혼잡 페널티
출구 근처 칸의 혼잡 감점 (β=0.3)

핵심 통찰: 중간 칸의 이중성

지하철 역은 플랫폼 중앙에 계단/에스컬레이터가 집중되어 있습니다. 중간 칸(3~7호차) 근처에서 하차하는 승객이 많아 자리가 빨리 남지만, 동시에 탑승하는 승객도 몰려 초기 착석이 어렵습니다. SeatScore v3는 이 트레이드오프를 β 페널티와 L(c) 경쟁 계수로 정밀하게 모델링합니다.

SeatScore 파라미터 튜닝

SeatScore v3 모델의 핵심 파라미터를 직접 조정하고, 추천 결과가 어떻게 변하는지 실시간으로 확인하세요.

β (탑승 혼잡 페널티 계수)

0.30
0.0 — 페널티 없음 1.0 — 강한 페널티

역할: 탑승 시 혼잡한 칸에 대한 감점 비율을 조절합니다.

β=0.0 탑승 혼잡 무시 → 하차 기회(Benefit)만 고려. 중간칸(4~6호차) 추천 많음
β=0.3 균형 설정 (기본값) → Benefit과 Penalty의 적절한 트레이드오프
β=0.7+ 혼잡 회피 우선 → 양끝칸(1,10호차)도 추천될 수 있음

💡 출퇴근 시간대에는 β를 높여 혼잡 회피를 강화하는 것이 효과적입니다.

시설 가중치 w(c,s)

역할: 역 내 하차 시설 유형별로 하차량 가중치를 부여합니다.

지하철역에서 에스컬레이터 근처에서 가장 많은 승객이 하차하며, 이는 착석 기회를 증가시킵니다.

가장 편리 → 하차 집중
교통약자/짐 많은 승객
기본값 (기준)

💡 가중치가 높을수록 해당 시설 근처 칸에서 착석 기회가 많다고 판단합니다.

시간대 배율 α(h)

역할: 시간대별로 전체 SeatScore에 곱해지는 배율입니다.

러시아워에는 승객이 많아 칸 선택의 중요도가 높아지고, 심야에는 어느 칸이든 착석 가능성이 높아 배율이 낮습니다.

⚡ 시스템은 이 기준값들 사이를 연속적으로 보간(interpolation)하여 부드러운 변화를 반영합니다.

가장 혼잡 → 칸 선택 중요
출근보다 약간 완화
기준값 (보통 혼잡도)
한산 → 칸 선택 덜 중요

💡 α > 1.0: 칸 선택 효과 증폭 | α < 1.0: 칸 선택 효과 감소

β 민감도 분석

β값을 0.0에서 1.0까지 변화시키며 칸별 점수 순위가 어떻게 뒤바뀌는지 시각화합니다. 먼저 추천 탭에서 경로를 선택한 뒤 분석을 실행하세요.

Metropy 연구 배경

8가지 공공데이터를 결합한 지하철 착석 효용 극대화 의사결정 모델 (SeatScore v3)

1. 문제 정의

수도권 지하철 이용에서 이용자가 체감하는 핵심 불편 요소 중 하나는 "앉을 수 있는가"입니다. 그러나 대부분의 지하철 정보 서비스는 혼잡도, 빠른 하차, 환승 효율 등 이동 이후의 편의성에 초점을 맞추고 있으며, 이동 중 착석 경험 자체는 직접 고려하지 않습니다.

"같은 열차를 탄다는 전제하에,
어느 차량(칸)에 탑승해야 이동 중 착석 효용이 최대화되는가?"

이것은 상태(state)를 예측하는 문제가 아니라, 출발 시점에서의 선택을 비교하는 의사결정(decision) 문제입니다.

2. 예측이 아닌 의사결정

칸별 착석률이나 칸별 하차 인원에 대한 정답 데이터(ground truth)는 존재하지 않습니다. 따라서 전통적인 지도학습 예측 문제로 접근할 수 없습니다.

Metropy는 이를 점수 기반 의사결정 문제로 재정의합니다. 8가지 관측 가능한 공공데이터(혼잡도, 역 구조, 시설 위치, 칸별 하차율, 실제 이동시간 등)를 결합한 효용 함수 SeatScore v3를 설계하여 칸 간의 상대적 착석 효용을 비교합니다.

4. 왜 2호선인가?

빠른환승 API(getFstExit) 테스트 결과, 2호선이 450개 레코드, 43개 역, 10량 고정 편성으로 모든 노선 중 가장 풍부하고 일관된 데이터를 보유하고 있었습니다. 순환선 특성상 내선/외선 양방향 분석이 가능하며, 수도권 최다 이용 노선으로 실용성이 높습니다.

5. 주요 발견

3호차 10개 시나리오 중 5개에서 1위
4호차 10개 시나리오 중 3개에서 1위
1, 10호차 양 끝 칸은 일관되게 최하위

중간 칸(3~7호차)은 출구 시설이 집중되어 하차 승객이 많고, 자리가 빨리 비지만, 동시에 탑승 인원도 몰립니다. 이 트레이드오프가 SeatScore v2의 핵심 모델링 대상입니다.

6. 한계 및 주의사항

  • 의사결정 지원 도구이지 착석 여부의 예측이 아닙니다. 정답(ground truth) 기반 검증은 구조적으로 불가능합니다.
  • D(s)는 30분 단위 역 전체 평균 혼잡도 기반이며, 실시간 차량 내 인원과는 차이가 있을 수 있습니다.
  • w(c,s)는 SK API 칸별 하차율 + 시설 위치 기반이며, 승객 개인 행동까지 반영하지는 않습니다.
  • β=0.3, α 기준값은 휴리스틱 설정이며, 파라미터 탭에서 민감도 분석으로 직접 검증할 수 있습니다.
  • 점수는 칸 간의 상대적 효용 비교이지, 절대적 착석 확률(%)을 의미하지 않습니다.
